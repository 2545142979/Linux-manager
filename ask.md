# 物联网监控系统技术规范文档 (V1.0)

本规范定义了基于 A9 处理器的智能监控系统的软件架构及通信协议。

## 1\. 系统架构规范

### 1.1 硬件组成

  * **采集端**：Camera（摄像头）、M0 传感器终端（通过 串口连接）。
  * **核心板**：A9 处理器（运行嵌入式 Linux，负责多线程调度）。
  * **客户端**：Qt 跨平台应用（通过网线/TCP 连接）。

### 1.2 软件线程模型 (A9 端)

| 线程编号 | 模块名称 | 核心逻辑描述 |
| :--- | :--- | :--- |
| **线程 1** | 视频采集线程 | 初始化摄像头 $\rightarrow$ 循环采集图像 $\rightarrow$ 存入全局结构体/缓冲区 $\rightarrow$ 异常处理。 |
| **线程 2** | 环境数据线程 | 初始化串口 $\rightarrow$ 循环读取 M0 数据 $\rightarrow$ 解析协议 $\rightarrow$ 更新全局结构体。 |
| **线程 3** | TCP 业务线程 | 监听客户端请求：<br>1. 收到 `"pic"`：从全局结构体读取最新图片发送。<br>2. 收到控制指令：解析指令并通过串口下发给 M0。 |

## 2\. 通信协议规范 (Protocol)

系统采用定长数据包格式，**包长度固定为 36 字节**。

### 2.1 帧头结构 (Header)

所有数据包的前 4 字节固定如下：

  * **Byte 0 (Tag)**: 帧头标识（`0xBB` 环境包, `0xDD` 命令包, `0xCC` 刷卡包）。
  * **Byte 1 (ID)**: 设备 ID（示例值：`0x08`）。
  * **Byte 2-3 (Len)**: 全包长度（小端存储，`0x24 00` 代表 36 字节）。

### 2.2 接收环境数据包 (Tag: 0xBB)

用于 A9 上报或 Qt 接收环境实时状态。

| 偏移 (Byte) | 字段 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| 4 - 5 | **tmp** | uint16 | 温度数据 |
| 6 - 7 | **hum** | uint16 | 湿度数据 |
| 8 - 11 | **acc** | uint32 | 三轴加速度数据 |
| 12 - 19 | **adc** | uint64 | ADC 原始值 |
| 20 - 23 | **light** | uint32 | 光照强度 |
| 24 - 27 | **state** | uint32 | 状态位 |
| 28 - 31 | **padding** | - | 字节对齐预留 |
| 32 - 35 | **crc** | uint32 | 循环冗余校验 |

### 2.3 发送控制命令包 (Tag: 0xDD)

用于 Qt 客户端下发指令。核心控制逻辑位于 **Byte 4 (Operation Code)**。

| 操作码 (Byte 4) | 动作标识 | 物理行为 |
| :---: | :--- | :--- |
| **0x00** | LED\_ON | 打开 LED 灯 |
| **0x01** | LED\_OFF | 关闭 LED 灯 |
| **0x02** | BUZ\_ON | 打开蜂鸣器 |
| **0x03** | BUZ\_OFF | 关闭蜂鸣器 |
| **0x04** | FAN\_ON | 打开风扇 |
| **0x08** | FAN\_OFF | 关闭风扇 |

## 3\. 交互逻辑总结

1.  **Upstream (上行)**: M0 $\xrightarrow{串口}$ A9 $\xrightarrow{TCP/IP}$ Qt (Tag: 0xBB)。
2.  **Downstream (下行)**: Qt $\xrightarrow{TCP/IP}$ A9 $\xrightarrow{串口}$ M0 (Tag: 0xDD)。
3.  **Media (多媒体)**: Qt 发送字符串 `"pic"` $\rightarrow$ A9 返回摄像头缓冲区数据。
